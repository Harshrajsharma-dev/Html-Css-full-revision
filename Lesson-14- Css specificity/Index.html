CSS Specificity & Cascade

1. What is CSS Specificity?

Specificity determines which CSS rule takes precedence when multiple rules apply to the same element.

  • Inline styles – highest specificity
  • ID selectors – 100 points
  • Class, attribute, pseudo-class selectors – 10 points
  • Element and pseudo-element selectors – 1 point

2. How to Calculate Specificity

#nav ul li a        → 100 + 1 + 1 + 1 = 103
.nav-link:hover    → 10 + 10 = 20
a                  → 1

Higher specificity wins. If equal, later rule in the source wins.

3. The Cascade: Four Stages

  1. Importance – !important overrides all
  2. Origin – Browser default → User → Author styles
  3. Specificity – Higher specificity wins
  4. Source order – Later styles override earlier ones

4. Selector Types & Examples

  • style="..." → Inline style
  • #header → ID selector
  • .btn, [type="text"], :hover → Class/attribute/pseudo-class
  • p, ::after → Element/pseudo-element

5. Common Conflicts

  • ID selectors override class selectors
  • Inline styles override stylesheet rules (unless overridden by !important)
  • Overly specific selectors lead to maintenance issues

6. Best Practices

  • Use low-specificity selectors (class > ID)
  • Avoid deep nesting and inline styles
  • Use !important only when truly needed
  • Keep CSS modular and readable

7. Cascade in Practice

Final style = Importance → Origin → Specificity → Source Order

Reordering rules can impact output—organize CSS thoughtfully.

✅ Summary

Specificity + Cascade + Source Order = Style that gets applied. Use class-based, maintainable CSS for cleaner code.
